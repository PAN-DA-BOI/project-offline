# Project Offline
Offline, decentralised communication that doesnt piggyback off of any other infrastructure is important to have in case of emergency, and for message security, and integrity.

### Why Not Meshtastic?
Meshtastic has out grown to become something that anyone could easily integrate into their project, creating lots of traffic on the networks, and people looking for exploits. We also wanted server side intigrations so that communication is never tied to a single device, and messages are saved in a secure location. Meshtastic is of course a great option, but it simply doesnt fit our needs.

## Getting Started
First, figure out what device your installing firmware or software too. Then go to the directory with that name, and review the READ-ME.

For example, if your installing firmware to a T-Deck Pro, then simply go to that directory and follow the READ-ME instructions there.
